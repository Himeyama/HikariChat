# HikariChat デザインガイドライン

## 16px グリッドシステム

このプロジェクトの UI デザインは、**16px グリッドシステム**を採用しています。

### 基本原則

- **すべての余白（margin, padding）は 16px の倍数を基準とする**
- **サイズ（width, height）は 16px の倍数を基準とする**
- **ギャップ（gap）は 16px の倍数を基準とする**

### 使用する値の例

```css
/* 推奨 - 16px 基準 */
margin: 16px;
margin: 32px;
margin: 48px;
padding: 16px;
padding: 24px;  /* 16 の倍数でないが使用可 */
gap: 16px;
gap: 24px;

/* 最小単位 - 8px の倍数まで許容 */
margin: 8px;    /* 最小単位 */
padding: 8px;   /* 最小単位 */
border-radius: 4px;  /* 細かい装飾は 4px も可 */
border-radius: 8px;

/* 非推奨 */
margin: 12px;
padding: 6px;
gap: 10px;
```

### 値の階層

| 用途 | 推奨値 |
|------|--------|
| 最小余白 | 8px |
| 標準余白 | 16px |
| 中余白 | 24px |
| 大余白 | 32px |
| 特大余白 | 48px, 64px |

### 例外

以下の場合は 16px 倍数でなくても良い：

- `1px` のボーダー
- `1.5` などの行間（line-height）
- フォントサイズ（em, rem 単位）
- 色の透明度（rgba のアルファ値）
- 細かい装飾（border-radius: 4px など）

### 理由

1. **視覚的な一貫性**: 16px 基準で揃えることで、UI 全体に明確なリズムと一貫性が生まれます
2. **十分な余白**: 16px を基準にすることで、十分な余白を確保し、視認性が向上します
3. **スケーラビリティ**: 高 DPI ディスプレイでのスケーリングが容易になります
4. **開発効率**: デザインの意思決定が迅速になり、実装も簡単になります
5. **アクセシビリティ**: 十分な余白とタッチターゲットサイズを確保できます

## 色の使用

### プライマリカラー

- `#0078d7` - Windows アクセントカラー（ユーザーアクション、リンク）

### セマンティックカラー

- `#f0f0f0` - 背景、ニュートラル
- `#ffe0e0` - エラー背景
- `#c00` - エラーテキスト
- `#333` - プライマリテキスト
- `#666` - セカンダリテキスト
- `#999` - プレースホルダー

### コードブロック

- `#2d2d2d` - コードブロック背景
- `#f8f8f2` - コードブロックテキスト
- `#e8e8e8` - インラインコード背景

## フォント

- システムフォントを使用
- コードは等幅フォント（Consolas, Monaco, Courier New）

## レスポンシブデザイン

- 最小タッチターゲットサイズ：40px × 40px
- ボタンの最小幅：96px（16px × 6）
- 入力フィールドの最小高さ：48px（16px × 3）

## コンポーネント規格

### ボタン

- 最小幅：96px
- 高さ：48px
- 左右パディング：24px
- 上下パディング：12px-16px

### 入力フィールド

- 高さ：48px
- パディング：12px 16px
- ラベルとの間隔：8px

### カード/コンテナ

- 内部パディング：24px
- 要素間ギャップ：16px

## チャットUI

- **メッセージ配置**: 人間（ユーザー）のメッセージは右側に、AI（アシスタント）のメッセージは左側に配置する。
- **メッセージ幅**: チャットバブルの最大幅はコンテナの70%とする。
